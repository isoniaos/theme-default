# v0.6 Address Component Theme Preparation

Status: preparation only. This note identifies where theme support for address
components should live. It does not finalize visual design.

## Current Theme Homes

- `src/theme.css` defines exported CSS variables with the `--iso-*` prefix.
- `src/tokens.ts` defines typed color, radius, spacing, typography, component,
  and layout tokens.
- `src/index.ts` exports the default theme module.

## State Coverage

Current color tokens already cover foreground, muted, border, primary, danger,
warning, success, and info surfaces. Address components should be prepared to
consume stable state names:

- neutral;
- success;
- warning;
- danger;
- accent.

For v0.6, `primary` can continue to act as the accent source until an explicit
accent alias is added. Neutral state styling should draw from foreground,
muted, border, surface, and surface-subtle tokens.

## Likely Token And Style Locations

- Add any new typed token names in `src/tokens.ts` before consuming them in App Core.
- Mirror any new token names as CSS variables in `src/theme.css`.
- Add component variant metadata to `ThemeComponentVariants` in `src/tokens.ts`
  only when App Core needs a stable theme contract for address display, address
  input, address chips, or copy affordances.
- Keep concrete address component CSS in App Core until the reusable public
  theme contract is clear.
