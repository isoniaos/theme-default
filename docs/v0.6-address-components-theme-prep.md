# v0.6 Address Component Theme Preparation

Status: foundation implemented. This note identifies where theme support for
address components lives. It does not finalize visual design.

## Current Theme Homes

- `src/theme.css` defines exported CSS variables with the `--iso-*` prefix.
- `src/tokens.ts` defines typed color, radius, spacing, typography, component,
  and layout tokens.
- `src/index.ts` exports the default theme module.

## State Coverage

Current color tokens already cover foreground, muted, border, primary, danger,
warning, success, and info surfaces. Address components should be prepared to
consume stable state names:

- neutral;
- success;
- warning;
- danger;
- accent.

For v0.6, `primary` can continue to act as the accent source until an explicit
accent alias is added. Neutral state styling should draw from foreground,
muted, border, surface, and surface-subtle tokens.

The default theme now exposes:

- `--iso-color-accent` and `--iso-color-accent-surface`;
- `--iso-address-avatar-size`, `--iso-address-avatar-radius`, and
  `--iso-address-letter-spacing`;
- reusable `.iso-state-neutral`, `.iso-state-success`, `.iso-state-warning`,
  `.iso-state-danger`, and `.iso-state-accent` classes that set
  `--iso-state-*` variables for components.

## Likely Token And Style Locations

- Add any future typed token names in `src/tokens.ts` before consuming them in
  App Core.
- Mirror any future token names as CSS variables in `src/theme.css`.
- Extend component variant metadata in `ThemeComponentVariants` only when App
  Core needs a stable theme contract beyond the address defaults now present.
- Keep concrete address component CSS in App Core until the reusable public
  theme contract is clear.
